Design Principle: Tests (specs) contain only test orchestration. All UI interactions go through Page Objects. All business operations go through Services. All WebDriverIO session management goes through DriverFactory.

2. Spec 1 — AppChef CLI Version Compatibility

Filetest/specs/appchef-version.spec.ts
Suite Timeout30 seconds
Test Count1
Page ObjectsNone
ServicesNone (direct HTTP)

Purpose: Ensures the latest wm-reactnative-cli version on GitHub is compatible with the version range declared in the AppChef build agent.

#Test CaseApproachAssertions
before Setup Fetches package.json from both GitHub repos in parallel (Promise.all). Extracts the CLI version and AppChef's declared dependency range. Validates the CLI version is valid semver. Fails fast if either HTTP request fails or the dependency entry is missing. —
1 should have the latest CLI version satisfy the AppChef dependency range Uses semver.satisfies(latestVersion, appchefRange) to verify compatibility. Asserts true. Error message includes both versions and specifies which repo needs an update.
after Teardown Logs suite completion. —

5. Spec 4 — Maven Build & Expo Go Verification

Filetest/specs/maven-expo.spec.ts
Suite Timeout30 minutes
Test Count10
Page ObjectsExpoGoAppPage
ServicesProjectService, MavenService, AppiumService

Purpose: End-to-end pipeline — downloads a WaveMaker project from Studio, builds the backend WAR, generates React Native ZIPs for both DEV and DEPLOYED backends, then verifies each app runs in Expo Go on Android and iOS.

#Test CaseApproachVerification
before Config & Appium Loads WMProjectConfig from .env. Logs all config (project ID, environment, server URLs). Starts Appium server via AppiumService. Appium server running.
1 should download and extract WaveMaker project from studio Uses ProjectService.downloadAndExtract():
(a) Authenticates via POST /login/authenticate
(b) Exports project via POST .../export with ZIP options
(c) Downloads ZIP via HTTP stream
(d) Extracts with adm-zip 		Extracted directory exists and contains pom.xml.
2 should build backend WAR file using mvn clean install Uses MavenService.cleanInstall() — runs mvn clean install, waits for "BUILD SUCCESS", scans target/ for .war file. WAR file exists on disk. Skips if project not downloaded.
3 should generate React Native ZIP with DEV server URL Uses MavenService.buildRnZip('dev', 'dev') — runs mvn clean package -Dmobile.serverUrl="dev", finds the ZIP, renames with -dev suffix. ZIP file exists with -dev suffix.
4 should generate React Native ZIP with DEPLOYED server URL Same flow with the deployed server URL and -deployed suffix. ZIP file exists with -deployed suffix.
5 should extract and prepare DEV React Native project MavenService.extractAndPrepareProject() — extracts ZIP, runs npm install. Extracted directory exists. Skips if ZIP unavailable.
6 should extract and prepare DEPLOYED React Native project Same as above for the DEPLOYED ZIP. Extracted directory exists.
7 should run and verify DEV app in Expo Go on Android runInExpoGo() helper:
(a) Appium session → Expo Go
(b) Spawns npm run android, waits for Metro ready
(c) 15s delay for app load
(d) ExpoGoAppPage.verifyAppRunning() 		Element visible within 60s. Screenshot on failure. SKIP_EXPO_GO
8 should run and verify DEPLOYED app in Expo Go on Android Same as test 7, using the DEPLOYED project (real backend URL). Element visible.
9 should run and verify DEV app in Expo Go on iOS simulator macOS only Opens iOS Simulator, waits 10s, then calls runInExpoGo() with XCUITest + host.exp.Exponent. iOS has a 20s post-startup wait (slower simulator). Element visible. Skips if not macOS.
10 should run and verify DEPLOYED app in Expo Go on iOS simulator Same as test 9, using the DEPLOYED project. Element visible.
after Cleanup & Summary Stops Appium server. Prints summary of all generated artifacts (WAR, DEV ZIP, DEPLOYED ZIP). —

7. Failure Handling Strategy

7.1 Screenshot on Failure

Every test with Appium or browser interaction captures a screenshot before rethrowing:

catch (error) {
  if (client) await DriverFactory.takeScreenshot(client, 'descriptive-name');
  throw error;
}

Screenshots are saved to allure-results/ with timestamped filenames and attached to Allure reports.

7.2 Graceful Dependency Skipping

Tests that depend on a prior test's output skip instead of failing:

if (!projectPath) {
  log.warn('Skipping: project not downloaded');
  this.skip();
}
Benefit: Only the root cause test is marked as failed. Dependent tests show as skipped, preventing a cascade of false failures in the report.
